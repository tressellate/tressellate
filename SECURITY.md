# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Tressellate, please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, email: security@bulKrete.com

We will acknowledge receipt within 48 hours and provide a detailed response within 7 days.

## Scope

Security issues in Layers 1 and 2 (hashgraph-tools and guardian-tools) are in scope. This includes:

- Authentication and authorization flaws
- Private key exposure risks
- Transaction signing vulnerabilities
- Input validation bypasses
- Dependency vulnerabilities

## Hiero Network Security

Issues with the Hiero network should be reported through the Hiero security disclosure process at [https://hiero.org/security](https://hiero.org/security), or to the Hedera team at [https://hedera.com/security](https://hedera.com/security).
